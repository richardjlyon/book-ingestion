## 1. Scope

M2.1 closes the gap between M2.0 (EPUB metadata extraction) and the full IR contract that the `ingesting-books` skill already consumes from the PDF backend. After M2.1, `book-ingest survey <epub>` and `book-ingest extract --chapter N <epub>` work the same way they do for PDFs, with the same exit codes, the same cache semantics, and the same `BookSurvey` / `ChapterContent` JSON shapes.

**In scope (M2.1).**

- New `EpubNativeBackend` in `src/book_ingestion/backends/epub_native.py`, implementing the existing `Backend` Protocol from `backends/base.py`.
- New chapter-reconciliation module `src/book_ingestion/structure/epub_chapters.py` (analogue of `structure/embedded_toc.py`).
- New XHTML→simple_view projection `src/book_ingestion/projection/epub_to_simple_view.py` (analogue of `projection/to_simple_view.py`).
- Extraction of shared OPF / zip / DRM helpers into `src/book_ingestion/extractors/_epub_common.py`. M2.0's `extractors/epub.py` is rewired to import from there; behavioural delta zero.
- `api.py._backend_for()` extended to dispatch `"epub"` to the new backend.
- New flags added to `KNOWN_FLAGS` in `quality/flags.py`; full list in §7.
- Three-layer test coverage matching the existing PDF pattern: unit (synthesised in-memory EPUBs), backend integration, real-EPUB acceptance fixtures (Pappé + one shape-diversity choice).

**Zero new runtime dependencies.** Stdlib `zipfile` + `xml.etree.ElementTree` — same path M2.0 uses today. No `[epub-ir]` extra (the base install handles EPUB IR; `[pdf-ir]` continues to gate Docling).

**Schema unchanged.** `SCHEMA_VERSION` stays `"1.1"`. `BlockType` unchanged. `Locator` discriminated union already includes `SpineRange` ([ir.py:42](../../../src/book_ingestion/ir.py:42)). All EPUB-specific behaviour lives in producers' choice of values for already-defined fields plus additive flag-vocabulary growth.

**Out of scope (deferred or other tools).**

- OCR for scanned PDFs (M3).
- MOBI / AZW3 / KF8 / other Amazon formats.
- DRM removal of any kind (detect-and-fail is the policy; tool never cracks DRM).
- EPUB authoring or modification (read-only tool).
- Skill-side adjustments to `ingesting-books/SKILL.md` (Cowork-side work; surfaced via `.cowork/inbox/0024-...`).
- New BlockType values for `blockquote` / `pre` / `image` (normalised into existing types per §3 decision; revisit only if downstream consumers actually demand the distinction).
- LLM-assisted EPUB chapter recovery.
- HTTP fetches of any kind from the EPUB path.

## 3. Design decisions

Each open question from §5 of the brief is resolved here. Decisions are locked.

### 3.1 `page_label` for EPUB blocks (brief §5.1)

**Decision: hybrid — real-or-None.**

For each block emitted from an EPUB:

- **If the EPUB carries print-page markers** — EPUB 3 `<nav epub:type="page-list">` entries, or in-content `<a id="page-N"/>` / `<a class="page" id="..."/>` anchors, or any other element with `epub:type="pagebreak"` plus a `title` or `id` carrying the printed number — `page_label = "<printed page number>"`. Real, reader-verifiable, matches what someone holding the print edition would see.
- **If the EPUB does not carry print-page markers** (Calibre conversions, born-digital EPUBs, anything that discarded print pagination) — `page_label = None`.

No synthesised pseudo-pages. No spine-index dressed up to look like a page number. The IR already accommodates `Paragraph.page_label: str | None` ([ir.py:100](../../../src/book_ingestion/ir.py:100)); we just stop pretending None is unusual.

The block-level `page` field carries the **spine index** (1-based, matching PDF's 1-based page convention). Chapter `locator` stays `SpineRange` (already in the IR).

**Skill-side ripple** (already surfaced via `.cowork/inbox/0024-tool-to-skill-m2.1-page-label-decision.md`):

- When `page_label` is a string, cite as today: `"chN, p X"`.
- When `page_label` is None on an EPUB-sourced block, cite at chapter granularity: `"chN"`. No `"p ?"`, no `"p None"`, no pseudo-page invention.

### 3.2 Chapter detection precedence (brief §5.2)

**Decision: nav-first → spine → headings, with deterministic fallbacks.**

Three signals are available inside an EPUB:

1. `nav.xhtml` (EPUB 3) or `toc.ncx` (EPUB 2) — the navigation table the reader sees. Closest analogue to the printed Contents page.
2. OPF `<spine>` — the canonical reading order.
3. `<h1>` / `<h2>` headings inside content XHTML — implicit structure.

The reconciliation recipe is a deterministic five-step procedure, fully specified in §5. Nav drives chapter assembly when usable; spine is the reconciliation target (handles multi-file chapters); headings are both a fallback and a splitter (chapter-spanning files). `MapInfo.provenance` honestly surfaces which path was taken so the skill's Step 0 reconciliation can spot-check.

### 3.3 Block-type extension (brief §5.3)

**Decision: normalise to existing BlockType values; do not extend the enum in M2.1.**

EPUBs may produce structural variants the PDF backend doesn't typically see: `<blockquote>`, `<pre>`/code, `<aside>`, `<figure>` with embedded images. Mapping for M2.1:

- `<blockquote>` → `Paragraph` (text content joined). The skill consumes paragraph text and at this stage does not distinguish quoted blocks from plain prose.
- `<pre>` / code → `Paragraph` with text content preserved verbatim. Trade non-fiction (the only acceptance-fixture genre) effectively never carries code.
- `<aside epub:type="footnote">` → `Footnote` (existing type).
- `<figure>` / `<figcaption>` → `FigureCaption` only when caption text exists; pure image figures are dropped.
- `<table>` → `Table` (existing type) with row extraction via cell innerText.
- `<a id="page-…">` / `<a class="page" id="…">` → consumed for `page_label`, not emitted as a block.

If a downstream consumer later requires the `blockquote` distinction (e.g. for ATX-style block-quote rendering in vault notes), extending `BlockType` is a follow-on milestone — the call should be made when there's evidence, not speculatively.

### 3.4 DRM detection (brief §5.4)

**Decision: honour M2.0's existing detection path; no new design.**

M2.0 already detects Adobe and Apple DRM markers in `META-INF/encryption.xml` ([extractors/epub.py:398](../../../src/book_ingestion/extractors/epub.py:398)). M2.1's `EpubNativeBackend.survey()` calls the shared `detect_drm()` helper (extracted into `_epub_common.py` per §3.7) before any further work. On detection, the backend raises `EpubDrmError`; the CLI translates to exit code 2 with a structured diagnostic (matching the `DRM_PROTECTED` error convention).

### 3.5 Cache schema version (brief §5.5)

**Decision: no bump. `SCHEMA_VERSION` stays at `"1.1"`.**

`BlockType`, block field shapes, `Locator` discriminated union, and top-level payload types are unchanged. Flag-vocabulary growth (§7) is purely additive — consumers should treat unknown flag strings as opaque. PDF cache entries stay valid. EPUB cache files (`survey.json`, `chapter-{idx}.json`, future `epub-parsed.json`) carry different artifact names from PDF (`docling.json`) so there is no collision risk on the same content-hash directory.

Reserved for the next bump: any change to `BlockType` enum values, any change to block field shapes, any change to `Locator` discriminator values.

### 3.6 EPUB parsing library (brief §4.1)

**Decision: continue stdlib `zipfile` + `xml.etree.ElementTree`. No new third-party dependency.**

Rationale:

- M2.0's metadata extractor already runs on this stack. Continuing it gives a single OPF-parsing implementation across both extractor and backend (via `_epub_common.py`).
- EPUB content XHTML is well-formed XML by spec — stdlib `ET.parse` handles it.
- nav.xhtml and NCX are XML — stdlib handles them.
- Adding `ebooklib` (~3 MB plus transitive deps) buys less bespoke code but doubles the parsing surface (M2.0 stays on stdlib unless we also rewrite it — out of scope).
- Real-world malformed-XHTML risk: handled per-content-file by emitting `failed_region` blocks rather than failing the whole extract. If real fixtures hit this often enough that lxml's looser parsing materially helps, adding lxml at that point is a fixture-driven follow-up (see "fixture-as-it-arises" in the project's working memory).

### 3.7 M2.0 OPF code reuse (brief §4.5)

**Decision: extract shared helpers to `extractors/_epub_common.py`; rewire M2.0's imports.**

The new module exposes:

- `open_epub_zip(path: Path) -> zipfile.ZipFile` — opens the zip with the same error-translation M2.0 uses today (`BadZipFile` / `OSError` → caller's choice of error).
- `find_opf_path(zf: zipfile.ZipFile) -> str | None` — currently `_find_opf_path` at [extractors/epub.py:204](../../../src/book_ingestion/extractors/epub.py:204). Moved verbatim, made non-private.
- `parse_opf_root(zf: zipfile.ZipFile, opf_path: str) -> ET.Element` — wraps the `ET.fromstring(zf.read(opf_path))` block currently inline at [extractors/epub.py:418](../../../src/book_ingestion/extractors/epub.py:418).
- `detect_drm(zf: zipfile.ZipFile, names: set[str]) -> bool` — encapsulates the Adobe/Apple namespace check at [extractors/epub.py:398](../../../src/book_ingestion/extractors/epub.py:398).
- `read_pageList_anchors(opf_root: ET.Element, zf: zipfile.ZipFile, names: set[str], opf_dir: str) -> dict[str, str]` — NEW helper; scans `<nav epub:type="page-list">` and in-content `<a class="page" id="page-N"/>` markers; returns `{anchor_id_or_href_fragment: printed_label}`.

M2.0's `EpubMetadataExtractor` is updated to import and call these instead of the now-removed local helpers. Behavioural delta zero — covered by the existing M2.0 test suite (`tests/test_metadata_epub.py`, `tests/test_metadata_real.py`) which must still pass unchanged after the move.

### 3.8 Quality scoring (brief §4.4)

**Decision: every successfully-parsed EPUB block carries `Confidence.EXCELLENT`. Failed parses emit `failed_region`. No middle gradations.**

Rationale: zip + XML parsing is deterministic. If the bytes parse, the block is correct — there is no parser confidence to expose. Inventing a heuristic (XHTML tag-cleanliness scoring) would mislead consumers who treat `confidence` as a parser-confidence signal. The IR's `confidence` field stays uniform across formats; EPUB just lives at the top of its dynamic range.

`Quality.docling_mean_grade` and `Quality.docling_low_grade` are `None` for EPUB surveys. `Quality.backend = "epub_native"`.

Broader concerns about the EPUB's own quality (broken nav, chapter-spanning files, missing pageList, parse failures) are surfaced via flags (§7), not via confidence gradations.

## 4. Public API surface

No new public functions. The existing `survey()` and `extract_chapter()` in `api.py` already accept any path; M2.1 wires the EPUB backend behind them.

```python
# src/book_ingestion/api.py
def _backend_for(path: Path) -> Backend:
    fmt = detect_format(path)
    if fmt == "pdf":
        if "pdf" not in _BACKENDS:
            from book_ingestion.backends.pdf_docling import PdfDoclingBackend
            _BACKENDS["pdf"] = PdfDoclingBackend()
        return _BACKENDS["pdf"]
    if fmt == "epub":  # NEW
        if "epub" not in _BACKENDS:
            from book_ingestion.backends.epub_native import EpubNativeBackend
            _BACKENDS["epub"] = EpubNativeBackend()
        return _BACKENDS["epub"]
    raise ValueError(f"no backend registered for format: {fmt}")
```

CLI surface in `cli.py` requires no changes. The `--no-cache`, `--cache-dir`, `--chapter N`, and exit-code (`0` / `2` / `1`) contracts are dispatch-agnostic and already work for any registered backend.

`extract_metadata()` is unchanged and continues to call `EpubMetadataExtractor` as today.

## 5. Chapter reconciliation recipe (decision §3.2 expanded)

The recipe runs inside `structure/epub_chapters.py:build_chapter_map_epub()`. Inputs and outputs:

```python
def build_chapter_map_epub(
    *,
    spine: list[SpineItem],          # [(idx_1based, href_in_zip, media_type), ...]
    nav: NavTree | None,             # parsed nav.xhtml or NCX, or None
    zf: zipfile.ZipFile,             # for opening content files when heading split needed
    opf_dir: str,                    # zip-internal dir of the OPF (for resolving hrefs)
) -> tuple[list[Chapter], MapInfo, set[str]]:
    """Returns (chapters, map_info, additional_flags)."""
```

`additional_flags` lets the recipe signal which path was taken (e.g. `nav_used`, `spine_only`, `headings_split_used`, `chapter_spans_multiple_files`) without the recipe importing the flag vocabulary directly. The backend assembles the final `quality.flags` list.

### Step 1 — Filter spine to content-XHTML items

Drop spine items whose manifest media-type is not `application/xhtml+xml` or `text/html`. Drops cover-as-image entries, drops nav.xhtml when listed in the spine. The remaining items are the corpus; their `(idx, href, media_type)` tuples are renumbered 1-based after filtering and that renumbering is the `page` value blocks carry.

### Step 2 — Choose the path

```
if nav exists AND ≥ 2 of nav's top-level entries resolve to filtered-spine items:
    → Step 3 (nav-driven, provenance=EMBEDDED, method="epub_nav")
elif nav exists but resolves to < 2 spine items, OR nav is absent:
    → Step 4 (spine-only, provenance=INFERRED, method="epub_spine") + flag spine_only
elif filtered spine is empty:
    → Step 5 (headings-only, provenance=INFERRED, method="epub_headings") + flag toc_unresolved
```

### Step 3 — Nav-driven assembly

For each top-level NavEntry in document order, with its `target_href` resolved to a `target_spine_idx`:

- **Clean one-file-per-chapter** (target_frag is None and next nav entry's target_spine_idx is target_spine_idx + 1): emit `Chapter(SpineRange(start=target, end=target))`.
- **Multi-file chapter** (target_frag is None and next entry's target_spine_idx > target + 1): emit `Chapter(SpineRange(start=target, end=next - 1))` and add flag `chapter_spans_multiple_files`.
- **Chapter starts mid-file** (target_frag is not None): emit `Chapter(SpineRange(start=target, end=end_spine, start_frag=target_frag, end_frag=next.target_frag))` where `end_spine` is determined by the next entry's spine target; add flag `headings_split_used`. The fragment IDs flow into `extract_chapter` for slicing within the file.

`Chapter.title` from the NavEntry text. `Chapter.provenance = EMBEDDED`. `Chapter.confidence = EXCELLENT`.

### Step 4 — Spine-only assembly

One chapter per filtered-spine item.
- `Chapter.title` = first `<h1>` or `<h2>` text inside the content XHTML, falling back to the spine item's filename stem (humanised: `"03_chapter_one.xhtml"` → `"chapter one"` → `"Chapter One"`).
- `Chapter.locator = SpineRange(start=idx, end=idx)`.
- `Chapter.provenance = INFERRED`.
- `Chapter.confidence = FAIR`.

### Step 5 — Headings-only assembly (last resort)

Walk every content-XHTML file referenced anywhere in the manifest, in manifest order. Treat each `<h1>` as a chapter boundary (anchor by file + element index). Synthesise `SpineRange` locators by mapping file→synthesised-spine-idx (deterministic order). This branch is rare and is best-effort; flag `toc_unresolved`.

### Determinism property

Same EPUB → same chapter list, every time. No hashing of timestamps, no random tie-breaks. The recipe is referentially transparent given (spine, nav, file contents).

## 6. XHTML → simple_view projection

Inside `projection/epub_to_simple_view.py`. The module exposes one entrypoint:

```python
def project_xhtml_to_blocks(
    *,
    xhtml_bytes: bytes,
    spine_idx: int,
    page_label_map: dict[str, str],   # anchor_id → printed_page_label
    start_frag: str | None = None,    # fragment to begin emission at, or None for "from top"
    end_frag: str | None = None,      # fragment to stop emission at (exclusive), or None for "to bottom"
) -> list[Block]:
```

**Element handling table** (document order, depth-first):

| XHTML element                                        | Emitted block / action                                       |
|------------------------------------------------------|--------------------------------------------------------------|
| `<h1>`–`<h6>` (excluding nav-suppressed)             | `Heading(level=N, text=…)`                                   |
| `<p>`                                                | `Paragraph(text=…)`                                          |
| `<blockquote>`                                       | `Paragraph(text=…)` (decision §3.3)                          |
| `<pre>`                                              | `Paragraph(text=… verbatim)` (decision §3.3)                 |
| `<aside epub:type="footnote">`                       | `Footnote(id=elem.id, text=…)`                               |
| `<figure>`/`<figcaption>` with text                  | `FigureCaption(text=…)`                                      |
| `<table>`                                            | `Table(rows=cell-innerText, raw_text=…)`                     |
| `<a id="page-N">`, `<a class="page">`, `<span epub:type="pagebreak">` | Consumed → updates current `page_label`; not emitted |
| `<nav>`, `<a class="hidden-print">`, elements with `epub:type="cover|titlepage|toc"` | Skipped (boilerplate) |
| Anything else                                        | Recurse into children                                        |

**Page-label tracking.** A running `current_page_label: str | None` is updated when a page-anchor element is encountered. Every emitted block stamps `page = spine_idx`, `page_label = current_page_label` (which is None until/unless a page anchor has been seen).

**Page breaks across spine items.** When `extract_chapter` walks multiple spine items for a multi-file chapter, the backend inserts a `PageBreak(page=next_spine_idx, page_label=initial_label_of_next_item)` between items — analogous to the PDF backend's page-break insertion at [projection/to_simple_view.py:70](../../../src/book_ingestion/projection/to_simple_view.py:70).

**Confidence.** Every block emitted from a successful XHTML parse: `Confidence.EXCELLENT`. The only path to a non-EXCELLENT block in EPUB is `failed_region`, emitted at the file level when `ET.parse` fails (§8.3).

## 7. Quality flag vocabulary

Additions to `KNOWN_FLAGS` in [`quality/flags.py`](../../../src/book_ingestion/quality/flags.py). Existing flags retained as-is.

**New flags (EPUB-only):**

| Flag                              | Emitted when                                                       |
|-----------------------------------|--------------------------------------------------------------------|
| `nav_used`                        | nav.xhtml or NCX drove chapter detection                           |
| `spine_only`                      | spine was the sole signal (nav broken or absent)                   |
| `headings_split_used`             | at least one chapter required heading-based splitting              |
| `chapter_spans_multiple_files`    | at least one chapter spans 2+ spine items                          |
| `xhtml_parse_failure`             | at least one content file failed to parse                          |

**Existing flags reused by EPUB (no semantic change):**

| Flag                              | Emitted when                                                       |
|-----------------------------------|--------------------------------------------------------------------|
| `page_labels_embedded`            | EPUB has a usable pageList nav or in-content page anchors          |
| `page_labels_unresolved`          | EPUB has no print-page markers                                     |
| `unparseable`                     | whole-document failure (malformed zip, missing OPF)                |
| `toc_unresolved`                  | reached Step 5 (headings-only) of the chapter recipe               |

EPUB does not emit `page_labels_inferred` — for EPUB, page labels are either present in the source or absent; there is no inference path. The PDF backend continues to emit only its existing flag set; nothing forced to change there.

## 8. Error handling

EPUB-side errors mirror M2.0 conventions where they exist; introduce new error codes only where needed.

### 8.1 DRM-protected EPUB

`EpubNativeBackend.survey()` calls `detect_drm()` immediately after opening the zip. On detection: raise `EpubDrmError` (new exception, defined in `backends/epub_native.py`). CLI catches this in the same place it catches PDF-side errors and emits a structured diagnostic with exit code `2`. (M2.0's `DRM_PROTECTED` `ErrorCode` is per-extractor; the IR-side surface uses an exception, matching the PDF backend's convention of raising on whole-document failures.)

### 8.2 Malformed zip / missing container.xml / missing OPF

Backend returns a `BookSurvey` with `chapters=[]`, `map.provenance=NONE`, `map.method="none"`, and `quality.flags=["unparseable"]`. CLI exits `2`.

### 8.3 Per-content-file XHTML parse failure

Backend emits a single `FailedRegion(page=spine_idx, page_label=None, reason="xhtml_parse_failure", raw_text=salvaged_text)` block in place of the file's normal projection. `salvaged_text` is best-effort: strip tags via regex, preserve text-node content. The chapter still returns successfully; the whole-survey `quality.flags` includes `xhtml_parse_failure`. Exit code `0` (the file is partially usable).

### 8.4 `extract_chapter` index out of range

`IndexError` raised by the backend, matching the PDF backend's existing behaviour at [backends/pdf_docling.py:262](../../../src/book_ingestion/backends/pdf_docling.py:262). CLI exits `2`.

### 8.5 Nav present but unresolvable

When nav exists but its targets do not resolve to any spine item (e.g. broken hrefs), the recipe falls through to Step 4 (spine-only) + flag `spine_only`. No error is raised — the survey still returns a usable chapter map.

## 9. Cache integration

The `Cache` from [`cache.py`](../../../src/book_ingestion/cache.py) is content-hash-keyed and format-agnostic. EPUB integrates with no changes to the cache module.

**Artifact names per content-hash directory** (no collision with PDF):

| Artifact                  | PDF backend         | EPUB backend                         |
|---------------------------|---------------------|--------------------------------------|
| Survey                    | `survey.json`       | `survey.json` (same — schema-uniform)|
| Chapter content           | `chapter-{N}.json`  | `chapter-{N}.json` (same)            |
| Backend-specific raw      | `docling.json`      | (none — re-parsing EPUB is cheap)    |
| Per-chapter raw slice     | `chapter-{N}.docling.json` | (none — same reasoning)        |
| Cache metadata            | `meta.json`         | `meta.json` (shared)                 |

EPUB does not persist a parsed-tree intermediate. Reparsing a 5–20 MB zip + XML walk is sub-second; caching it adds invalidation surface for negligible benefit. Cache hits on `survey.json` and `chapter-{N}.json` skip all parsing.

## 10. Testing strategy

Three layers, mirroring the M2.0 / PDF pattern.

### 10.1 Unit tests (synthesised in-memory EPUBs)

New test files:

- `tests/test_epub_common.py` — `find_opf_path`, `parse_opf_root`, `detect_drm`, `read_pageList_anchors`. Existing M2.0 tests for these (`tests/test_metadata_epub.py`) continue to cover them via the rewired imports — these new tests target the helpers directly.
- `tests/test_epub_chapters.py` — `build_chapter_map_epub` recipe branches:
  - clean one-file-per-chapter → nav-driven path
  - multi-file chapter → flag `chapter_spans_multiple_files`
  - chapter starts mid-file (nav target carries fragment) → flag `headings_split_used`
  - nav resolves to < 2 spine items → spine-only fallback + flag `spine_only`
  - nav absent → spine-only fallback
  - filtered spine empty → headings-only last resort
- `tests/test_epub_projection.py` — `project_xhtml_to_blocks`:
  - h1–h6 → `Heading` with correct level
  - `<p>` and `<blockquote>` both → `Paragraph`
  - `<table>` rows extracted
  - `<aside epub:type="footnote">` → `Footnote`
  - page anchors update current `page_label` and are not emitted
  - boilerplate (`<nav>`, `epub:type="cover"`) skipped
  - fragment-bounded slice (start_frag / end_frag) honoured

Synthesised EPUBs are built in-test via `zipfile.ZipFile` writing OPF + nav + XHTML strings to `BytesIO`. Helper module `tests/_epub_fixtures.py` exposes a small builder DSL (`make_epub(spine=…, nav=…, content=…) -> bytes`) so tests stay declarative.

### 10.2 Backend integration tests

- `tests/test_epub_backend_survey.py` — full `EpubNativeBackend.survey()` against synthesised EPUBs:
  - all flag emissions verified
  - page_label stamping (pageList present vs absent)
  - cache hit / miss + schema_version round-trip
  - DRM marker → `EpubDrmError`
  - malformed zip → `unparseable` flag survey

### 10.4 Cross-format symmetry tests

## 12. Acceptance criteria
